
# Raspberry Pi 網路設定與管理筆記

## 1. 使用 Netplan 設定 Wi-Fi

編輯設定檔：

```bash
sudo nano /etc/netplan/50-cloud-init.yaml
```

範例：

```yaml
network:
  version: 2
  wifis:
    wlan0:
      dhcp4: true
      access-points:
        "MyWiFi":
          password: "MyPassword"
```

套用設定：

```bash
sudo netplan generate
sudo netplan apply
```

檢查設定：

```bash
sudo netplan status
```

## 2. 檢查網路狀態

### 查看網卡狀態

```bash
ip addr show wlan0
```

若顯示：

```text
3: wlan0: <NO-CARRIER,BROADCAST,MULTICAST,UP>
```

表示：

* 網卡已啟動（UP）
* 驅動正常
* 尚未連上 Wi-Fi AP（NO-CARRIER）
* 尚未取得 IP 位址

### 查看 IP 位址

```bash
hostname -I
```

或

```bash
ip addr
```

### 查看路由資訊

```bash
ip route
```

### 測試外網連線

```bash
ping -c 4 8.8.8.8
```

### 測試 DNS

```bash
ping -c 4 google.com
```

## 3. 安裝常用網路管理工具

更新套件：

```bash
sudo apt update
```

安裝工具：

```bash
sudo apt install -y \
network-manager \
wireless-tools \
iw \
net-tools \
avahi-daemon
```

安裝後可使用：

| 工具           | 功能                  |
| ------------ | ------------------- |
| nmcli        | NetworkManager 指令介面 |
| nmtui        | 文字圖形化網路管理介面         |
| iw           | Wi-Fi 除錯工具          |
| ifconfig     | 查看網路資訊              |
| netstat      | 查看網路連線              |
| avahi-daemon | 支援 *.local 網域名稱     |

## 4. 啟用 NetworkManager

啟用服務：

```bash
sudo systemctl enable NetworkManager
sudo systemctl start NetworkManager
```

確認狀態：

```bash
systemctl status NetworkManager
```

查看裝置：

```bash
nmcli device status
```

> **常見問題排除：**
> 
> **1. 顯示 `unmanaged` (不受管理)**
> 代表網路目前由 Netplan 管理。
> 1. 編輯 Netplan 設定檔：
>    ```bash
>    sudo nano /etc/netplan/50-cloud-init.yaml
>    ```
> 2. 在 `network:` 區塊下新增 `renderer: NetworkManager`：
>    ```yaml
>    network:
>      renderer: NetworkManager
>    ```
> 3. 套用設定並重啟服務：
>    ```bash
>    sudo netplan apply
>    sudo systemctl restart NetworkManager
>    ```
> 
> **2. 顯示 `unavailable` (無法使用)**
> 代表介面被停用或認證服務衝突。請依序執行：
> - `sudo nmcli radio wifi on` (開啟射頻)
> - `sudo ip link set wlan0 up` (喚醒網卡)
> - 若依然無效，通常是 `wpa_supplicant` 卡死，請直接執行 `sudo reboot` 重新開機。

## 5. 使用 nmcli 管理 Wi-Fi

### 掃描 Wi-Fi

```bash
nmcli device wifi list
```

> **注意：如果掃描不到預期的網路**
> 1. **快取問題**：`nmcli` 有時會顯示舊的快取結果。若需強制重新掃描，請執行以下指令（務必加 `sudo` 以免出現 `not authorized` 錯誤）：
>    ```bash
>    sudo nmcli device wifi rescan
>    ```
>    執行後請稍待 3~5 秒，再重新使用 `nmcli device wifi list` 即可看到更新後的清單。
> 2. **硬體限制**：若為較舊型號的樹莓派（如 Pi 3），其內建網卡僅支援 2.4GHz 頻段，無法掃描或連線到 5GHz 的 Wi-Fi 網路。

### 連接 Wi-Fi

```bash
nmcli device wifi connect "SSID名稱" password "密碼"
```

### 查看已儲存連線

```bash
nmcli connection show
```

### 查看目前連線

```bash
nmcli device status
```

### 確保自動連接開啟

```bash
sudo nmcli connection modify "您的WiFi名稱" connection.autoconnect yes
```

## 6. 使用 nmtui（推薦）

啟動：

```bash
sudo nmtui
```

功能：

* 掃描 Wi-Fi
* 選擇 AP
* 輸入密碼
* 管理已儲存連線
* 設定固定 IP
* 修改主機名稱

適合透過 SSH 管理網路。

## 7. 使用 iw 進行 Wi-Fi 除錯

### 查看目前連線

```bash
iw dev wlan0 link
```

### 掃描附近 AP

```bash
sudo iw dev wlan0 scan | grep SSID
```

### 查看無線資訊

```bash
iw dev
```

## 10. 常見問題

### 顯示 NO-CARRIER

```text
wlan0: <NO-CARRIER,BROADCAST,MULTICAST,UP>
```

可能原因：

* Wi-Fi 未設定
* SSID 輸入錯誤
* 密碼錯誤
* AP 訊號太弱
* DHCP 失敗
* NetworkManager 未啟動

### 已連上 SSH

若已能透過 SSH 連線：

```bash
ssh username@IP
```

表示：

* Wi-Fi 已成功連線
* DHCP 已取得 IP
* SSH 服務正常
* 網路功能正常運作
